Projects

Reverse engineering of Tor's key agreement protocol
As part of my student thesis, I reverse engineered the “Tor Rendezvous Specification - Version 3” so I could take inspiration for my own key agreement protocol. This was done purely on code basis, as the informal description was lacking detail in many regards. As part of that, I translated the informal description of the key agreement given on the TOR project page into an formal cryptographic one.
AN.ON Client Application Extension
Design and implementation of recipient anonymity on the client side of AN.ON. Includes the design of a new key agreement protocol, with inspiration taken from TOR’s key agreement protocol. Uses a custom version of the Bouncy Castle crypto library. Encryption is done via AES-128 CTR, signatures are beeing generated via Ed25519 and Curve25519 is used as the underlying key agreement mechanism.
BraillePlot
A Java-based tool to create Braille embosser text files for generic diagram representations, which then can be printed on an Braille embosser. To ensure extensibility and variability of the Software, the architecture was designed with these corresponding patterns in mind, so that generic Braille printers and embossers can be used interchangably. A visualization tool (also in Java) was created so that the user has a preview option of the Braille text which will be printed.
Risk assessment and evaluation of ORAN
As part of a student thesis, I identified, evaluated and prioritized risks of an ORAN-Infrastructure for the typical UE. As a result, I successfully found high-priority risks in both ORAN and the “normal” RAN architecture defined by the 3GPP and I compared these against each other.
Towards protection of availability in 5G-Networks
My Diploma Thesis, which was about the protection of availability in 5G-Networks. The goal was to estimate, if the 3GPP standards could be misused in such a way, that an insider, in form of a misbehaving 5G core component, could leverage attacks against the availability of the 5G-Network.
Simulator for interplanetary Internet
Project collaboration on a Java and C++ based simulator for interplanetary internet, which includes the bundle protocol for Delay Tolerant Networking.
AN.ON Server Application Extension
Design and implementation of recipient anonymity on the server side of AN.ON. Although the client is written in Java, the server (for performance reasons) is purely written in C. Uses a custom version of OpenSSL for encryption, signatures and key agreement mechanisms.
